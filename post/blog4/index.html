使用hugo和GitHub搭建个人博客

Dec 12, 2020
1 min read

搭建步骤

安装brew

安装git

安装hugo

命令行 brew install hugo

新建文件夹

在想建立站点的位置新建一个文件夹（不要出现中文路径）
如 blog

打开终端

1.cd blog   打开文件夹
2.hugo new site myblog 新建名为myblog的站点
3.通过theme.github.io 下载主题并配置
    下载
    cd myblog
    git clone https://主题路径
    配置
    主题文件夹下有exampleSite文件夹
    打开文件夹，找到theme.toml 配置好后更改文件名为config.toml （注意备份）
    复制上面的文件到根目录myblog
    ps. 主题文件夹下的static为放置图片的文件夹，头像等图片资源一般都放在这个文件夹
4.hugo new post/blog.md     在content/post 文件夹中新建一个后缀名为md的博客文件
5.编辑博客文件
6.hugo server -D     在本地运行测试

GitHub 操作

新建个人仓库，名称为用户名+github.io  如dushiyu1.github.io

终端操作

1.hugo --theme=m10c --baseUrl="https://dushiyu1.github.io" -buildDrafts  
此时会在myblog文件夹内生产public文件夹
2.cd public  打开public文件夹
3.git init   初始化git仓库
4.git add .  添加文件
5.git commit -m”提交说明”    提交文件
7.git remote add origin https://github.com/dushiyu1/dushiyu1.github.io.git
部署文件夹到远程仓库
8.git push   推送到远程仓库
9. Git  push -u origin master 如第8条不可用，使用该条命令

此时GitHub中仓库内已上传，通过仓库域名即可访问网页。

关于更新博客

1.hugo new post/blog.md  新建一个博客文件 
2.编辑blog并保存
3.hugo  -server -D. 本地测试
4.hugo   更新public文件信息
5.cd public   打开public文件夹
6.git add .   添加文件
7.git commit -m”提交更新说明”  提交文件
8.git push     推送到远程仓库
此时GitHub中源代码更新，通过仓库域名即可看到更新后的网页。
